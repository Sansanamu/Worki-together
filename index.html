생성형 AI 프로젝트 협업 공간

문제 정의

소규모 IT 프로젝트(졸작·캡스톤·사이드 프로젝트 등)에서 팀원들이 생성형 AI를 각자 사용하는 과정에서 AI 출력의 일관성 부족, 개인 설정·메모리로 인한 문맥 혼재, 팀 단위 기록 분산 문제가 발생했습니다. 프로젝트의 목표는 팀 전용 AI 협업 환경을 구축해 문맥·톤·작업 기준을 통일하고, 동시에 문서·회의 기록을 한 공간에서 관리할 수 있는 워크스페이스를 제공하는 것이었습니다.

아키텍처 / 설계

  • React 기반 단일 페이지 웹 애플리케이션(SPA)으로 개발
  • Node.js + Express를 사용하여 API 서버 구현
  • OpenAI API를 활용해 팀 단위 LLM 컨텍스트 및 역할별 에이전트 구성
  • Supabase를 통해 프로젝트 문서·회의 기록 저장 및 인증 처리
  • 팀 워크스페이스 형태의 문서 보관 UI 설계
  • 팀별 프로젝트 설정(룰셋, 프롬프트)을 독립적으로 관리하는 데이터 스키마 구성

핵심 기능

1. 팀 전용 LLM 컨텍스트 관리

팀마다 독립적인 ‘프로젝트 맥락’을 서버에서 생성/저장하고, 프로젝트 개요, 기술 스택, 룰셋 등 구조화된 데이터를 LLM 호출 시마다 자동 포함시켜 팀원 모두가 동일한 문맥 기반으로 일관된 결과물을 생성하도록 설계했습니다.

2. 역할 기반 AI 어시스턴트

PM / FE / BE / Designer 등 포지션별 시스템 프롬프트를 사전에 정의하고, 프로젝트 설정값과 역할별 맥락을 조합해 목적에 맞는 결과물을 생성합니다. (예: PM은 요구사항 중심, FE는 React/UX 중심)

3. 문서·회의록 보관소

프로젝트 문서(PRD, 기획서, API 스펙 등)를 태그 기반 분류 및 버전 기록 기능과 함께 저장하는 공간을 제공합니다. 단일 문서 저장 구조를 통해 정보 분산을 최소화하고 회의 내용도 한 곳에서 관리합니다.

4. 작업 로그 및 결정사항 보드

팀의 작업 로그와 회의 중 결정 사항을 보드에서 관리합니다. 진행 중/보류/완료 등 상태 기반으로 이슈를 정리하여, 메신저나 구두 소통으로 분산되는 내용을 프로젝트 공간 내부로 통합합니다.

나의 역할 및 기여

  • 프로젝트 기획, 문제 정의, 핵심 기능 선정 등 전체 PM 역할 담당
  • 팀 단위 LLM 컨텍스트 설계 및 역할 기반 시스템 프롬프트 구성
  • React 기반 구조 및 문서 보관소 UI/UX 설계
  • Node.js 및 OpenAI API 연동을 통한 AI 기능 프로토타입 구현
  • Supabase를 활용해 인증·데이터 스키마·문서 저장 구조 구축

결과 및 성과

  • 팀 단위 LLM 환경 구성으로 AI 출력 일관성 향상 (정성적 확인)
  • 프로젝트 문서·회의 기록 중앙화로 기록 탐색 시간 약 30~40% 감소
  • 개발자·디자이너·PM 역할별 AI 어시스턴트로 프로토타입 단계 생산성 개선
  • 초기 사용자 테스트(팀 내부)에서 "정보 분산 감소/정리 용이함" 긍정 피드백 확보
